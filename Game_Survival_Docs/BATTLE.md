# 전투 시스템 설계

---

## 개요

탑다운 뷰에서 근접 / 원거리 무기로 적(Enemy)을 처치.
플레이어는 HUD 버튼 또는 적 터치로 공격하며, 공격 타이밍에 범위 밖에 있으면 자동 회피 처리.

---

## 플레이어 전투

| 항목 | 결정 |
|------|------|
| 시점 | 탑다운 |
| 공격 방식 | HUD 공격 버튼 (가장 가까운 적 자동 타겟) + 적 터치 동시 지원 |
| 자동 공격 | 보류 |
| 무기 종류 | 근접 / 원거리 |
| 스태미나 | 보류 |
| 회피 | 공격 타이밍에 공격 범위 밖에 있으면 자동 회피 (스태미나 소모 없음) |
| 피격 | HP 감소 |
| 크리티컬 | 보류 |

### 공격 조작 — 두 가지 동시 지원

| 방식 | 설명 |
|------|------|
| HUD 공격 버튼 | 범위 내 가장 가까운 적을 자동 타겟, 버튼 탭으로 공격 |
| 적 터치 | 터치한 적을 직접 지정하여 공격 |

- 두 방식 모두 동일한 공격 처리 로직 사용 (`TryAttack(Enemy target)`)
- 공격 버튼은 범위 내 적이 있을 때만 활성화 (HarvestButton과 동일 패턴)

### 회피 판정

```
적이 공격 실행
  └─ 플레이어가 공격 범위 내에 있는가?
       ├─ Yes → 피격 처리 (HP 감소)
       └─ No  → 회피 처리 (데미지 없음)
```

- 적의 공격 범위는 테이블(`EnemyData`)로 관리
- 플레이어가 이동 중이면 자연스럽게 회피 가능

### 무기 분류

| 종류 | 특성 |
|------|------|
| 근접 | 짧은 사거리, 빠른 공격속도 |
| 원거리 | 긴 사거리, 탄약 소모 |
| 도구 (근접 겸용) | 채집 도구로도 사용 가능 |

### 데미지 계산

> 추후 확정 예정. 현재는 `공격력 - 방어력` 단순 구조로 임시 구현.

```
최종 데미지 = max(공격력 - 방어력, 1)
```

## 적 (Enemy)

> 세계관 미정 — 코드상 `Enemy`로 통일, 명칭은 추후 변경

### 적 종류

| 타입 | 특성 |
|------|------|
| 기본형 | 느리고 약함, 근접 공격 |
| 고속형 | 빠름, 약한 공격 |
| 탱커형 | 느리고 HP 높음, 강한 공격 |

### 적 AI — FSM

```
Idle (배회)
  └─ 탐지 범위 내 플레이어 감지 OR 범위 내 동료 피격 알림 수신
       └─ Chase (추격)
            └─ 공격 사거리 진입
                 └─ Attack (공격)
                 │    └─ 플레이어 사거리 이탈 → Chase 복귀
                 └─ Dead (사망)
                      └─ 드롭 처리 후 제거
```

| 항목 | 결정 |
|------|------|
| 탐지 방식 | 시야 반경 (EnemyData 테이블로 적별 관리) |
| 군집 반응 | 피격 시 주변 일정 반경 내 동료에게 알림 → 전부 Chase 전환 |
| 피격 반응 | 피격 시 추격 시작 (시야 밖이어도) |
| 이동 | NavMesh 기반 |

### 군집 반응 상세

```
Enemy A 피격
  └─ Physics.OverlapSphere(피격 위치, AlertRadius)
       └─ 범위 내 Enemy → Chase 상태로 전환
```

- `AlertRadius`는 `EnemyData` 테이블에서 관리
- 연쇄 반응 없음 (B가 알림 받아 이동 중 피격돼도 추가 전파 안 함)

## 사망 처리

| 대상 | 처리 |
|------|------|
| 적 사망 | 인벤토리 자동 추가 (ResourceHarvested와 동일 패턴), 일정 시간 후 오브젝트 제거 |
| 플레이어 사망 | 추후 결정 |

## 데이터 테이블 구조

### EnemyData

| 필드 | 타입 | 설명 |
|------|------|------|
| `enemy_id` | int | 적 고유 ID |
| `name` | string | 적 이름 |
| `hp_max` | int | 최대 HP |
| `attack` | int | 공격력 |
| `defense` | int | 방어력 |
| `move_speed` | float | 이동 속도 |
| `detect_radius` | float | 탐지 반경 |
| `attack_radius` | float | 공격 반경 |
| `alert_radius` | float | 군집 반응 알림 반경 |
| `attack_interval` | float | 공격 간격 (초) |
| `drop_item_id` | int | 드롭 아이템 ID |
| `drop_count_min` | int | 최소 드롭 수량 |
| `drop_count_max` | int | 최대 드롭 수량 |
| `prefab_path` | string | 프리팹 경로 (Resources 기준) |

## 구현 순서

1. `EnemyData` 데이터 클래스 + `Enemy.json` 샘플 데이터
2. `IDamageable` 인터페이스 (Player / Enemy 공통)
3. `Enemy` — HP 관리, FSM (Idle/Chase/Attack/Dead), NavMesh 이동
4. `EnemySpawner` — 테이블 기반 스폰 (ResourceSpawner와 동일 패턴)
5. `MainPanel` — 공격 버튼 추가 (범위 내 적 감지 시 활성화)
6. 적 터치 입력 처리
7. 드롭 처리 (EnemyDeadEvent → 인벤토리 자동 추가)
