# 인벤토리 시스템 설계

---

## 개요

플레이어가 소지하는 아이템을 관리하는 시스템.
슬롯 + 무게 이중 제한으로 자원 관리 긴장감 부여.

---

## 구조

| 항목 | 결정 |
|------|------|
| 제한 방식 | 슬롯 수 + 무게 이중 제한 |
| 슬롯 수 | **미정** |
| 최대 무게 | **미정** |
| 장비 슬롯 | 무기 / 방어구 / 도구 (별도 슬롯) |
| 일반 슬롯 | 자원, 아이템 소지 |

---

## 아이템 분류

| 분류 | 스택 | 설명 |
|------|------|------|
| 자원류 | ✅ 가능 | 나무, 돌 등 재료 |
| 소비류 | ✅ 가능 | 음식, 치료제 |
| 장비류 | ❌ 불가 | 무기, 방어구, 도구 |
| 특수류 | ❌ 불가 | 퀘스트 아이템, 희귀 부품 |

---

## 베이스 저장 상자

- 인벤토리와 **완전 분리**
- 베이스 내 상자에 아이템 보관
- 상자 티어에 따라 슬롯 수 차이 — **미정**
- 탐험 시 인벤토리만 가져감

---

## 아이템 데이터 구조 (코드 설계 참고)

```
ItemData
  ├── item_id       (int)
  ├── name          (string)
  ├── description   (string)
  ├── category      (string) — Resource / Consumable / Equipment / Special
  ├── weight        (float)
  ├── max_stack     (int)    — 1이면 스택 불가
  ├── icon_path     (string)
  └── [장비 전용]
      ├── equipment_slot    — Weapon / Armor / Tool
      ├── durability_max    (int)
      └── tier              (int)
```

---

## 미결 사항

- [ ] 인벤토리 슬롯 수
- [ ] 최대 무게 수치
- [ ] 장비 슬롯 구성 상세 (방어구 부위 수 등)
- [ ] 상자 티어 및 슬롯 수
