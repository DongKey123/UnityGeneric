# 전투 시스템 설계

> 리니지M 전투 구조를 참조하여 설계하였습니다.

---

## 코드 구조 개요

```
IDamageable (인터페이스)
    ├── PlayerController
    └── MonsterController

PlayerController
    ├── PlayerStat          — 최종 스탯 계산 (레벨 + 장비 + 강화 합산)
    ├── StateMachine        — 대기 / 전투 / 사망
    ├── TargetingSystem     — 주변 탐색, 가장 가까운 타겟 선택
    ├── AttackSystem        — 기본 공격 타이머
    └── SkillSystem         — 스킬 슬롯 3개, 쿨타임 관리
```

## 이동 방식

**NavMesh** 사용 (확정)

- `NavMeshAgent` 컴포넌트로 이동/추격 처리
- 장애물 회피 자동 처리
- 쿼터뷰 + 단순 필드 구조에 적합
- 플레이어/몬스터 모두 동일하게 적용

---

## IDamageable

플레이어와 몬스터가 공통으로 구현하는 피해 처리 인터페이스입니다.

```csharp
public interface IDamageable
{
    bool IsDead { get; }
    void TakeDamage(int damage);
}
```

---

## PlayerStat

레벨 / 장비 / 능력치 강화 / 문신을 합산한 **최종 스탯**을 제공합니다.
스탯 변경 시 `OnStatChanged` 이벤트로 UI에 알립니다.

| 스탯 | 구성 |
|------|------|
| `MaxHp` | 기본(레벨) + 투구 + 갑옷 + 능력치강화 |
| `Atk` | 기본(레벨) + 무기 + 능력치강화 |
| `Def` | 기본(레벨) + 갑옷 + 장갑 + 신발 + 능력치강화 |
| `AtkSpeed` | 기본 1.0 + 장갑 + 능력치강화 |
| `AtkRange` | 무기 고정값 |
| `MoveSpeed` | 기본 3.0 + 신발 + 능력치강화 |
| `CritRate` | 기본 5% + 능력치강화 + 문신 |
| `CritDmg` | 기본 2.0배 + 능력치강화 + 문신 |

## 자동 / 수동 모드

| 모드 | 이동 | 공격 우선순위 | 타겟 |
|------|------|--------------|------|
| **자동** | 타겟 자동 추격 | 스킬 우선 → 기본 공격 | 자동 |
| **수동** | 조이스틱 / WASD 직접 조작 | 탐지 + 정지 상태일 때만 발동 | 자동 (탐지 범위 내) |

- HUD 버튼으로 언제든 전환 가능
- 수동에서 몬스터 탭/클릭 시 해당 몬스터로 타겟 변경

---

## 타겟팅 (탐지)

- `Physics.OverlapSphere(반경 15m)` 로 주변 몬스터 탐색 — 반경은 상수, **플레이어 기준 고정**
- 탐지 범위 내 **가장 가까운** 몬스터를 자동 선택
- 타겟 몬스터 머리 위 **타겟 마커** 표시
- 탐지 범위 내 몬스터가 없으면 → 대기 상태 (IdleState)
- 타겟 사망 시 → 다음 가장 가까운 몬스터로 자동 전환

---

## 자동 모드 전투 흐름

```
탐지
  없음 → 대기 (IdleState / 대기 상태)
  있음 →
    쿨타임 완료 스킬 있음 → 스킬 우선 발동
    쿨타임 완료 스킬 없음 → 기본 공격
      사정거리 내 → 멈추고 공격
      사정거리 밖 → 타겟 추격 → 사정거리 도달 → 멈추고 공격
```

- 이동 중 공격/스킬 불가 — 사정거리 도달 후 멈추고 발동 (리니지M 방식)

---

## 수동 모드 전투 흐름

```
조이스틱/WASD 입력 있음
  → 이동만 (공격/스킬 발동 없음)

조이스틱/WASD 입력 없음 (정지 상태)
  탐지 없음
    → 대기. 스킬 버튼 눌러도 발동 안 함
  탐지 있음
    사정거리 내 → 멈추고 스킬/기본 공격
    사정거리 밖 → 사정거리까지 자동 이동 → 도달 후 스킬/기본 공격
```

---

## 기본 공격

- 공격 간격: `1 / AtkSpeed` 초
- 이동 중 공격 불가 — 반드시 정지 후 발동

---

## 스킬 시스템

- 스킬 슬롯 3개, 각자 독립 쿨타임
- 자동: 쿨타임 완료 시 기본 공격보다 **우선** 자동 발동 (스킬1 > 스킬2 > 스킬3)
- 수동: 탐지 없으면 버튼 눌러도 발동 안 함. 탐지 있으면 사정거리 이동 후 발동
- 스킬 시전 중 이동 불가 (시전 시간 0.5초 고정)

| 스킬 | 쿨타임 | 타입 | 효과 |
|------|--------|------|------|
| 스킬1 | 5초 | 단일 | ATK × 2.0 |
| 스킬2 | 10초 | 범위 (반경 3m) | ATK × 1.5 |
| 스킬3 | 20초 | 단일 | ATK × 4.0 |

## 드롭 처리

```
몬스터 사망
  → 드롭 아이템(골드/경험치 오브젝트) 필드에 생성 (이펙트)
  → N초 대기
  → 플레이어를 향해 자동으로 날아옴 (이펙트)
  → 플레이어 도착 → 골드/경험치 즉시 지급
```

- 드롭 오브젝트는 ObjectPool로 관리
- 자동 수거 대기 시간: 미정 (2~3초 예상)

---

## 레벨업

```
경험치 획득
  → 누적 경험치 >= 필요 경험치 (LevelData 기준)
  → 레벨업
    → 최대 HP 증가 + HP 전체 회복
    → 레벨업 이펙트 출력
    → ToastManager 알림
  → 스탯 재계산 (PlayerStat.Recalculate)
```

> 스탯 설계 상세: [STAT_DESIGN.md](STAT_DESIGN.md) ← 별도 문서 작성 예정

## 스폰 시스템

```
StageManager
  → 챕터 데이터 기준 최대 몬스터 수 유지
  → 몬스터 사망 감지
    → N초 후 리스폰
  → 항상 최대 몬스터 수를 채운 상태 유지
```

- 최대 동시 몬스터 수: `ChapterData.max_monster_count`
- 리스폰 대기 시간: `ChapterData.respawn_delay`
- 몬스터 탐지 범위: `MonsterData.detect_range` (몬스터마다 개별 설정)

**스폰 위치 결정 방식:**
```
스테이지 중심 기준 spawn_radius 내 랜덤 위치 선정
  → 플레이어와의 거리 < spawn_min_distance_from_player 이면 재시도
  → NavMesh 위 유효한 위치인지 SamplePosition으로 검증
  → 최대 10회 재시도 후 실패 시 스킵 (다음 틱에 재시도)
```

## 스테이지 / 챕터 전환 방식

| 전환 | 방식 | 이유 |
|------|------|------|
| **스테이지 전환** (같은 챕터 내) | 씬 유지 — 몬스터 교체만 수행 | 배경/맵 변경 없음, 로딩 불필요 |
| **챕터 전환** | 씬 재로드 + 로딩 화면 | 배경/텍스쳐 교체 필요 |

- `StageManager.LoadStage(stageId)` — 기존 몬스터 풀 반환 후 새 스폰 데이터 적용
- 챕터 전환 시 `SceneLoader`로 씬 교체

## 피해 계산

```
기본 피해 = ATK - (DEF × 0.5)
최소 피해 = ATK × 0.1          ← 방어력이 높아도 최소 피해 보장
스킬 피해 = 기본 피해 × 스킬 배율
치명타   = 기본 피해 × CritDmg  ← CritRate 확률로 발동
```

## 캐릭터 상태 (StateMachine)

플레이어는 항상 아래 3가지 상태 중 하나로 동작합니다.

```
[대기 상태 / IdleState]
  감지 범위 내 몬스터 있음 → [전투 상태 / CombatState]

[전투 상태 / CombatState]
  타겟 사거리 밖 → 타겟 추격 이동
  타겟 사거리 내 → 멈춤 + 기본 공격
  스킬 쿨타임 완료 → 스킬 발동 (이동 중단)
  타겟 사망 → 다음 타겟 자동 선택 (없으면 → [대기 상태])
  내 HP 0 → [사망 상태 / DeadState]

[사망 상태 / DeadState]
  사망 연출 (0.5초)
  3초 대기
  HP 전체 회복 → [대기 상태]
```

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| 대기 상태 `IdleState` | 범위 내 몬스터 감지 → 전투 상태 | 주변 탐색 반복 |
| 전투 상태 `CombatState` | HP 0 → 사망 상태 / 몬스터 없음 → 대기 상태 | 추격 + 공격 + 스킬 |
| 사망 상태 `DeadState` | 3초 후 자동 부활 → 대기 상태 | 사망 연출 → HP 회복 |

---

## 몬스터 상태 (StateMachine)

```
[대기 상태 / IdleState]
  스폰 후 0.5초 대기 → [추격 상태 / ChaseState]

[추격 상태 / ChaseState]
  플레이어를 향해 이동
  AtkRange 이내 진입 → [공격 상태 / AttackState]

[공격 상태 / AttackState]
  공격 쿨타임마다 플레이어 공격
  플레이어가 AtkRange 밖으로 이탈 → [추격 상태]
  HP 0 → [사망 상태 / DeadState]

[사망 상태 / DeadState]
  사망 연출 (0.5초)
  드롭 처리 (골드 / 경험치)
  ObjectPool 반환
```

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| 대기 상태 `IdleState` | 스폰 후 0.5초 → 추격 상태 | 스폰 연출 |
| 추격 상태 `ChaseState` | AtkRange 진입 → 공격 상태 | 플레이어를 향해 이동 |
| 공격 상태 `AttackState` | 이탈 → 추격 상태 / HP 0 → 사망 상태 | 쿨타임마다 공격 |
| 사망 상태 `DeadState` | 연출 완료 → ObjectPool 반환 | 사망 애니메이션 + 드롭 |
