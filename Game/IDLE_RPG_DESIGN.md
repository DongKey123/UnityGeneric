# 방치형 RPG — 게임 기획서

Unity Generic Framework 샘플 프로젝트입니다.
프레임워크의 주요 모듈 연동을 검증하는 것이 목적입니다.

---

## 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 방치형 RPG |
| 레퍼런스 | 리니지M, 뮤 오리진 |
| 화면 방향 | 세로 (Portrait) |
| 시점 | 3D 쿼터뷰 (카메라 고정 등각 투영) |
| 플랫폼 | 모바일 (Android / iOS) |
| 핵심 루프 | 필드 자동 사냥 → 골드/경험치 획득 → 강화/레벨업 → 더 강한 필드 진입 |

### 게임 특징

- 캐릭터가 필드에서 몬스터를 자동으로 타겟, 이동, 공격
- 자동 스킬 사용 (쿨타임마다 자동 발동)
- 앱을 끄고 있어도 오프라인 보상 지급 (방치 보상)
- 스테이지 / 필드 개념 — 구역마다 몬스터 종류와 난이도 상이
- 성장 콘텐츠 — 레벨업, 장비 강화, 스킬 강화

---

## 핵심 루프

### 매크로 루프 (큰 그림)

플레이어가 장기적으로 반복하는 성장 사이클입니다.
"조금만 더 강해지면 저 필드 가야지"라는 동기를 유지하는 것이 핵심입니다.

```
필드 자동 사냥
    ↓
골드 / 경험치 / 장비 획득
    ↓
레벨업 / 장비 강화 / 스킬 강화
    ↓
더 강한 필드 진입
    ↓
(반복)
```

### 마이크로 루프 (전투 안)

전투 중 매 순간 반복되는 사이클입니다.
플레이어 개입 없이 자동으로 진행됩니다.

```
범위 내 몬스터 자동 타겟
    ↓
몬스터를 향해 자동 이동
    ↓
사거리 도달 → 기본 공격 자동 발동
    ↓
스킬 쿨타임 완료 → 스킬 자동 발동
    ↓
몬스터 사망 → 골드/경험치 드롭
    ↓
다음 몬스터 자동 타겟
    ↓
(반복)
```

### 방치 루프 (오프라인)

앱을 종료해도 보상이 쌓이는 방치형 핵심 메커니즘입니다.
재접속 동기를 부여하고 꾸준한 플레이를 유도합니다.

```
앱 종료 (자동 사냥 중이던 필드 기준으로 보상 계산)
    ↓
오프라인 경과 시간 × 시간당 골드/경험치
    ↓
앱 재실행 → 오프라인 보상 수령 팝업
    ↓
보상으로 강화 → 더 강한 필드 진입
```

> 오프라인 보상은 최대 12시간치까지만 적립됩니다.

## 세계관

삼국지를 모티브로 한 3개 나라가 대립하는 세계입니다.

| 나라 | 특징 | 미정 |
|------|------|------|
| 나라 A | 미정 | 이름/종족/특성 추후 결정 |
| 나라 B | 미정 | 이름/종족/특성 추후 결정 |
| 나라 C | 미정 | 이름/종족/특성 추후 결정 |

- 플레이어는 게임 시작 시 나라 1개 선택
- 나라별 스토리/필드/전용 컨텐츠 차별화 예정

> **추후 컨텐츠** — 진영 전쟁, 점령전, 나라별 PvP, 진영 보상 등은 추후 기획 예정

## 필드 / 챕터 구성

### 구조

```
챕터 1 (초보자의 땅)   ← 시작부터 개방
챕터 2 (미정)          ← 레벨 20 + 챕터1 몬스터 500마리 처치
챕터 3 (미정)          ← 레벨 50 + 챕터2 몬스터 1000마리 처치
```

- 챕터 안에서 무한 자동 사냥
- 챕터 이탈 시 로비로 복귀

### 챕터 해금 조건

| 챕터 | 레벨 조건 | 처치 수 조건 |
|------|-----------|-------------|
| 챕터 1 | 없음 | 없음 |
| 챕터 2 | 레벨 20 이상 | 챕터 1 몬스터 500마리 |
| 챕터 3 | 레벨 50 이상 | 챕터 2 몬스터 1000마리 |

### 몬스터 구성

- 챕터별 등장 몬스터는 **데이터 테이블로 관리**
- 챕터마다 N종류 몬스터 설정 가능
- 스폰 비율도 테이블에서 설정 (예: 몬스터A 60% / 몬스터B 30% / 몬스터C 10%)

```json
// ChapterData.json 예시
{
  "chapter_id": 1,
  "name": "초보자의 땅",
  "unlock_level": 0,
  "unlock_kill_count": 0,
  "monsters": [
    { "monster_id": 1001, "spawn_weight": 60 },
    { "monster_id": 1002, "spawn_weight": 30 },
    { "monster_id": 1003, "spawn_weight": 10 }
  ]
}
```

### 드롭

- 챕터가 높을수록 골드 / 경험치 / 스탯 포인트 재화 드롭량 증가
- 드롭량도 데이터 테이블에서 관리

### 추후 컨텐츠

| 컨텐츠 | 설명 |
|--------|------|
| 보스타임 | 특정 시간대 보스 등장, 전 서버 공동 참여 |
| 진영 전쟁 | 3개 나라 간 필드 점령전 |
| 던전 | 스탯 포인트 재화 대량 획득처 |

## 캐릭터 성장

### 기본 스탯

| 스탯 | 설명 | 영향 요소 |
|------|------|-----------|
| `MaxHp` | 최대 체력 | 레벨 / 투구 / 갑옷 / 능력치 강화 |
| `Atk` | 공격력 | 레벨 / 무기 / 능력치 강화 |
| `Def` | 방어력 | 레벨 / 갑옷 / 장갑 / 신발 / 능력치 강화 |
| `AtkSpeed` | 공격 속도 | 장갑 / 능력치 강화 |
| `AtkRange` | 공격 사거리 | 무기 고정값 |
| `MoveSpeed` | 이동 속도 | 신발 / 능력치 강화 |
| `CritRate` | 치명타 확률 | 능력치 강화 / 문신 |
| `CritDmg` | 치명타 피해 배율 | 능력치 강화 / 문신 |

---

### 레벨업

- 경험치 획득 → 필요 경험치 도달 시 자동 레벨업
- 레벨업마다 MaxHp / Atk / Def 증가
- 레벨업 시 HP 전체 회복
- 레벨업 시 ToastManager 알림

---

### 장비

**슬롯 구성**

| 슬롯 | 영향 스탯 |
|------|-----------|
| 무기 | ATK, AtkRange |
| 투구 | MaxHp |
| 갑옷 | MaxHp, DEF |
| 장갑 | DEF, AtkSpeed |
| 신발 | DEF, MoveSpeed |

**장비 강화**

- 강화 단계: +0 ~ +10
- 재화: 골드
- 강화 비용: `기본 비용 × (현재 강화 단계 + 1)`
- 강화 효과: 단계당 기본 스탯 10% 추가

**장비 각인 (도감)**

- 보유한 장비를 파괴하여 도감에 등록
- 등록된 장비의 스탯을 영구 보유 (장비 없이도 스탯 유지)
- 같은 장비를 중복 등록하면 스탯 추가 누적
- 도감 등록은 강화 단계에 비례하여 스탯 기록 (+10 등록 시 최대 스탯)

```
예시)
Iron Sword +5 각인 → ATK +10 영구 보유
Iron Sword +10 각인 → ATK +20 영구 보유 (추가 누적)
```

---

### 문신

- 별도 슬롯에 장착하여 추가 스탯 부여
- 장비 시스템과 완전히 독립
- 슬롯 수: 5슬롯 (추후 확장 가능)

**등급**

| 등급 | 색상 | 스탯 수치 |
|------|------|-----------|
| 일반 | 흰색 | 낮음 |
| 희귀 | 파랑 | 중간 |
| 영웅 | 보라 | 높음 |
| 전설 | 주황 | 최고 |

**획득 방식**

- 몬스터 드롭 (낮은 확률)
- 합성: 같은 등급 문신 3개 → 상위 등급 1개
- 추후 추가될 컨텐츠에서 획득

**영향 스탯**

- ATK, MaxHp, DEF, CritRate, CritDmg

---

### 능력치 강화

- 골드를 소모하여 개별 스탯을 영구 강화
- 강화할수록 비용 증가
- 강화 항목: MaxHp / ATK / DEF / AtkSpeed / CritRate / CritDmg / MoveSpeed

---

### 스탯 포인트 (스테이지 한정)

- 전용 재화 소모 → 스탯 포인트 획득
- 스테이지 진행 중에만 적용되는 임시 강화
- 영구 스탯과 별개 — 스테이지 이탈 시 적용 유지, 재배분 시 재화 소모

**재화 획득처**

- 스테이지 몬스터 드롭 (스테이지 높을수록 많이 드롭)
- 추후 추가될 컨텐츠 (던전, 이벤트 등)에서 대량 획득

**배분 가능 스탯**

| 스탯 | 포인트당 증가량 |
|------|----------------|
| ATK | +1% |
| MaxHp | +1% |
| DEF | +1% |
| AtkSpeed | +0.5% |
| CritRate | +0.3% |

---

### 스킬

- 스킬 슬롯 3개
- 골드로 스킬 배우기 / 강화
- 스킬 강화마다 피해 배율 증가

| 스킬 | 배우기 비용 | 쿨타임 | 기본 배율 | 최대 강화 |
|------|-------------|--------|-----------|-----------|
| 스킬1 | 1,000G | 5초 | ATK × 2.0 | +10 (ATK × 4.0) |
| 스킬2 | 5,000G | 10초 | ATK × 1.5 (범위) | +10 (ATK × 3.0) |
| 스킬3 | 20,000G | 20초 | ATK × 4.0 | +10 (ATK × 8.0) |

## 전투 시스템

### 자동 / 수동 모드

| 모드 | 이동 | 공격 | 스킬 | 타겟 |
|------|------|------|------|------|
| **자동** | 가장 가까운 몬스터 자동 추격 | 자동 | 자동 (쿨타임마다) | 자동 |
| **수동** | 조이스틱 직접 조작 | 자동 | 자동 (쿨타임마다) | 자동 |

- HUD 우측 상단 **자동/수동 토글 버튼**으로 전환
- 수동 모드에서 조이스틱을 놓으면 현재 위치에서 제자리 공격
- 수동 모드에서 몬스터 탭 시 해당 몬스터로 타겟 변경

### 자동 모드 흐름

```
자동 모드 ON
  → 범위 내 가장 가까운 몬스터 자동 타겟
  → 몬스터를 향해 자동 이동
  → 사거리 도달 → 기본 공격 자동 발동
  → 스킬 쿨타임 완료 → 스킬 자동 발동
  → 몬스터 사망 → 드롭 → 다음 몬스터 자동 타겟
  → (반복)
```

### 수동 모드 흐름

```
수동 모드 ON
  → 조이스틱으로 캐릭터 이동
  → 이동 중 사거리 내 몬스터 자동 공격
  → 스킬 쿨타임 완료 → 스킬 자동 발동
  → 조이스틱 중립 → 제자리에서 가장 가까운 몬스터 공격
```

### 타겟팅

- 자동: 범위 내 **가장 가까운** 몬스터 자동 선택
- 수동: 몬스터 탭 시 해당 몬스터로 타겟 변경, 사망 시 가장 가까운 몬스터로 자동 전환
- 타겟 몬스터 머리 위에 **타겟 마커** 표시

### 기본 공격

- 타겟이 사거리 내에 있으면 자동 발동
- 공격 간격: `1 / AtkSpeed` 초
- 타겟이 사거리 밖이면 자동으로 추격 (자동 모드) / 제자리 대기 (수동 모드)

### 스킬

- 스킬 슬롯 3개 (스킬1 / 스킬2 / 스킬3)
- 쿨타임마다 자동 발동 (우선순위: 스킬1 > 스킬2 > 스킬3)
- 모든 스킬은 현재 타겟에게 발동

| 스킬 | 쿨타임 | 효과 |
|------|--------|------|
| 스킬1 | 5초 | 단일 대상 강타 (ATK * 2.0) |
| 스킬2 | 10초 | 범위 공격 (ATK * 1.5, 반경 3m) |
| 스킬3 | 20초 | 단일 대상 강력 일격 (ATK * 4.0) |

### 피해 계산

```
기본 피해 = ATK - (DEF * 0.5)
최소 피해 = ATK * 0.1   (방어력이 높아도 최소 피해 보장)
스킬 피해 = 기본 피해 * 스킬 배율
치명타 피해 = 기본 피해 * 2.0 (치명타 확률: 기본 5%)
```

### 캐릭터 StateMachine

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| `IdleState` | 범위 내 몬스터 감지 → CombatState | 대기 |
| `CombatState` | HP 0 → DeadState / 몬스터 없음 → IdleState | 타겟 추격 + 공격 + 스킬 |
| `DeadState` | 3초 후 → IdleState | 사망 연출 → HP 전체 회복 → 부활 |

### 몬스터 StateMachine

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| `IdleState` | 스폰 후 0.5초 → ChaseState | 스폰 연출 |
| `ChaseState` | 사거리 내 진입 → AttackState | 캐릭터를 향해 이동 |
| `AttackState` | 캐릭터 사거리 이탈 → ChaseState / HP 0 → DeadState | 공격 쿨타임마다 공격 |
| `DeadState` | 사망 연출 완료 → ObjectPool 반환 | 사망 애니메이션 → 드롭 처리 |
