# 방치형 RPG — 게임 기획서

Unity Generic Framework 샘플 프로젝트입니다.
프레임워크의 주요 모듈 연동을 검증하는 것이 목적입니다.

---

## 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 방치형 RPG |
| 레퍼런스 | 리니지M, 뮤 오리진 |
| 화면 방향 | 세로 (Portrait) |
| 시점 | 3D 쿼터뷰 (카메라 고정 등각 투영) |
| 플랫폼 | 모바일 (Android / iOS) |
| 핵심 루프 | 필드 자동 사냥 → 골드/경험치 획득 → 강화/레벨업 → 더 강한 필드 진입 |

### 게임 특징

- 캐릭터가 필드에서 몬스터를 자동으로 타겟, 이동, 공격
- 자동 스킬 사용 (쿨타임마다 자동 발동)
- 앱을 끄고 있어도 오프라인 보상 지급 (방치 보상)
- 스테이지 / 필드 개념 — 구역마다 몬스터 종류와 난이도 상이
- 성장 콘텐츠 — 레벨업, 장비 강화, 스킬 강화

---

## 핵심 루프

### 매크로 루프 (큰 그림)

플레이어가 장기적으로 반복하는 성장 사이클입니다.
"조금만 더 강해지면 저 필드 가야지"라는 동기를 유지하는 것이 핵심입니다.

```
필드 자동 사냥
    ↓
골드 / 경험치 / 장비 획득
    ↓
레벨업 / 장비 강화 / 스킬 강화
    ↓
더 강한 필드 진입
    ↓
(반복)
```

### 마이크로 루프 (전투 안)

전투 중 매 순간 반복되는 사이클입니다.
플레이어 개입 없이 자동으로 진행됩니다.

```
범위 내 몬스터 자동 타겟
    ↓
몬스터를 향해 자동 이동
    ↓
사거리 도달 → 기본 공격 자동 발동
    ↓
스킬 쿨타임 완료 → 스킬 자동 발동
    ↓
몬스터 사망 → 골드/경험치 드롭
    ↓
다음 몬스터 자동 타겟
    ↓
(반복)
```

### 방치 루프 (오프라인)

앱을 종료해도 보상이 쌓이는 방치형 핵심 메커니즘입니다.
재접속 동기를 부여하고 꾸준한 플레이를 유도합니다.

```
앱 종료 (자동 사냥 중이던 필드 기준으로 보상 계산)
    ↓
오프라인 경과 시간 × 시간당 골드/경험치
    ↓
앱 재실행 → 오프라인 보상 수령 팝업
    ↓
보상으로 강화 → 더 강한 필드 진입
```

> 오프라인 보상은 최대 12시간치까지만 적립됩니다.

## 캐릭터

### 기본 스탯

| 스탯 | 설명 |
|------|------|
| HP | 최대 체력 |
| ATK | 공격력 |
| DEF | 방어력 |
| ATK_SPD | 공격 속도 |

### 레벨업

- 경험치 획득 → 레벨업 → 스탯 증가
- 레벨업 시 ToastManager로 알림

### 장비

- 무기 / 방어구 2슬롯
- 골드로 강화 (+1 ~ +10)
- 강화 수치에 따라 스탯 증가

## 전투 시스템

### 자동 / 수동 모드

| 모드 | 이동 | 공격 | 스킬 | 타겟 |
|------|------|------|------|------|
| **자동** | 가장 가까운 몬스터 자동 추격 | 자동 | 자동 (쿨타임마다) | 자동 |
| **수동** | 조이스틱 직접 조작 | 자동 | 자동 (쿨타임마다) | 자동 |

- HUD 우측 상단 **자동/수동 토글 버튼**으로 전환
- 수동 모드에서 조이스틱을 놓으면 현재 위치에서 제자리 공격
- 수동 모드에서 몬스터 탭 시 해당 몬스터로 타겟 변경

### 자동 모드 흐름

```
자동 모드 ON
  → 범위 내 가장 가까운 몬스터 자동 타겟
  → 몬스터를 향해 자동 이동
  → 사거리 도달 → 기본 공격 자동 발동
  → 스킬 쿨타임 완료 → 스킬 자동 발동
  → 몬스터 사망 → 드롭 → 다음 몬스터 자동 타겟
  → (반복)
```

### 수동 모드 흐름

```
수동 모드 ON
  → 조이스틱으로 캐릭터 이동
  → 이동 중 사거리 내 몬스터 자동 공격
  → 스킬 쿨타임 완료 → 스킬 자동 발동
  → 조이스틱 중립 → 제자리에서 가장 가까운 몬스터 공격
```

### 타겟팅

- 자동: 범위 내 **가장 가까운** 몬스터 자동 선택
- 수동: 몬스터 탭 시 해당 몬스터로 타겟 변경, 사망 시 가장 가까운 몬스터로 자동 전환
- 타겟 몬스터 머리 위에 **타겟 마커** 표시

### 기본 공격

- 타겟이 사거리 내에 있으면 자동 발동
- 공격 간격: `1 / AtkSpeed` 초
- 타겟이 사거리 밖이면 자동으로 추격 (자동 모드) / 제자리 대기 (수동 모드)

### 스킬

- 스킬 슬롯 3개 (스킬1 / 스킬2 / 스킬3)
- 쿨타임마다 자동 발동 (우선순위: 스킬1 > 스킬2 > 스킬3)
- 모든 스킬은 현재 타겟에게 발동

| 스킬 | 쿨타임 | 효과 |
|------|--------|------|
| 스킬1 | 5초 | 단일 대상 강타 (ATK * 2.0) |
| 스킬2 | 10초 | 범위 공격 (ATK * 1.5, 반경 3m) |
| 스킬3 | 20초 | 단일 대상 강력 일격 (ATK * 4.0) |

### 피해 계산

```
기본 피해 = ATK - (DEF * 0.5)
최소 피해 = ATK * 0.1   (방어력이 높아도 최소 피해 보장)
스킬 피해 = 기본 피해 * 스킬 배율
치명타 피해 = 기본 피해 * 2.0 (치명타 확률: 기본 5%)
```

### 캐릭터 StateMachine

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| `IdleState` | 범위 내 몬스터 감지 → CombatState | 대기 |
| `CombatState` | HP 0 → DeadState / 몬스터 없음 → IdleState | 타겟 추격 + 공격 + 스킬 |
| `DeadState` | 3초 후 → IdleState | 사망 연출 → HP 전체 회복 → 부활 |

### 몬스터 StateMachine

| 상태 | 전환 조건 | 동작 |
|------|-----------|------|
| `IdleState` | 스폰 후 0.5초 → ChaseState | 스폰 연출 |
| `ChaseState` | 사거리 내 진입 → AttackState | 캐릭터를 향해 이동 |
| `AttackState` | 캐릭터 사거리 이탈 → ChaseState / HP 0 → DeadState | 공격 쿨타임마다 공격 |
| `DeadState` | 사망 연출 완료 → ObjectPool 반환 | 사망 애니메이션 → 드롭 처리 |

## 데이터

### JSON 데이터 (InGameDataManager)

| 파일 | 키 | 설명 |
|------|----|------|
| `StageData.json` | stage_id | 스테이지 정보 (몬스터 종류, 수, 추천 전투력) |
| `MonsterData.json` | monster_id | 몬스터 스탯 |
| `ItemData.json` | item_id | 장비 기본 스탯 |
| `LevelData.json` | level | 레벨별 필요 경험치, 스탯 증가량 |

### 저장 데이터 (SaveSystem)

| 키 | 내용 |
|----|------|
| `player_level` | 현재 레벨 |
| `player_exp` | 현재 경험치 |
| `player_gold` | 보유 골드 |
| `equipment` | 장비 강화 수치 |
| `current_stage` | 현재 스테이지 |

## 활용 모듈 목록

| 모듈 | 사용처 |
|------|--------|
| `StateMachine` | 캐릭터 / 몬스터 상태 관리 |
| `ObjectPool` | 몬스터, 공격 이펙트, 드롭 아이템 |
| `EventBus` | 레벨업, 스테이지 클리어, 몬스터 사망 이벤트 |
| `AudioManager` | BGM(로비/전투), SFX(공격/레벨업/강화) |
| `SaveSystem` | 플레이어 데이터 저장/로드 |
| `InGameDataManager` | 스테이지/몬스터/아이템/레벨 데이터 로드 |
| `SceneLoader` | Boot → Lobby → Battle 씬 전환 |
| `InputManager` | 모바일 터치 입력 |
| `UIManager` | HUD, 로비 패널 관리 |
| `CommonPopupManager` | 강화 확인, 일시정지 팝업 |
| `ToastManager` | 레벨업, 강화 결과 알림 |
| `SafeAreaFitter` | 노치/홈 인디케이터 회피 |
| `LocalizationSystem` | 한국어/영어 지원 |

## 디렉토리 구조 (예정)

```
Assets/
├── Scripts/
│   ├── Framework/       ← 기존 프레임워크
│   └── Game/
│       ├── Boot/        ← BootScene 초기화
│       ├── Lobby/       ← 로비 로직
│       ├── Battle/      ← 전투 로직
│       │   ├── Character/
│       │   ├── Monster/
│       │   └── Stage/
│       ├── Data/        ← 데이터 클래스 (StageData, MonsterData 등)
│       └── UI/          ← 게임 전용 UI 패널
├── Prefabs/
│   ├── Characters/
│   ├── Monsters/
│   └── UI/
├── Resources/
│   ├── Data/            ← JSON 데이터
│   ├── Localization/    ← 다국어 JSON
│   └── UI/              ← UI 프리팹
└── Scenes/
    ├── BootScene
    ├── LobbyScene
    └── BattleScene
```

## 개발 순서 (예정)

1. BootScene — 데이터 로드, 씬 전환

3. BattleScene — 캐릭터, 몬스터, 전투 로직
4. LobbyScene — 강화, 캐릭터 정보
5. UI 연동
6. 저장/로드
7. 오디오
8. 다국어
